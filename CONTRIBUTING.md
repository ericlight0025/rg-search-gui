# Contributing

Thanks for considering a contribution.

## Scope

This project is intentionally small and practical.
Please prefer focused improvements over broad framework changes.

## Before Opening a PR

- Confirm the change supports the main search workflow
- Avoid breaking the Windows MVP flow
- Keep changes easy to test and easy to roll back
- Do not commit generated cache files such as `__pycache__` or `*.pyc`

## Development

```bash
pip install -e .
python -m rg_search_gui
```

## Pull Requests

A good PR for this project should include:

- A clear problem statement
- A minimal implementation
- A short verification note
- Screenshots or logs when UI behavior changes
